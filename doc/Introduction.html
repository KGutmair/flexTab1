Introduction

library(flexTab1)

Introduction

This function creates a highly flexible summary measure table for descriptive statistics (“Table 1”) with options to customize both the statistical options and the output layout. It supports following options:

  • Group stratification: Supports summary measured for a single group (group_var = FALSE) and stratified by two or more groups. Additionally, a nested group structure is supported, allowing for the comparison of subgroups (group_var) within main groups treatment_arm(e.g. compare the subgroups x and y within the main groups A and B)

  • Statistical specifications:

    • measure_cat, measure_num: specifies, which summary measures should be displayed. For categorial variables, one can choose between the combination of “absolute” and “relative” frequency. For numerical variables, one can choose between any combination of “mean”, “median”, standard deviation “sd”, minimum “min”, maximum “max”, first quartile “quartile1”, third quartile “quartile3”.

    • display_missings: specifies, whether the absolute/relative frequencies of the missing values for each variable should be displayed.

    • display_pvalues: specifies, whether the distribution of baseline variables between groups should be tested for differences. Its p-values are displayed in the output table. Until now, only a comparison between two groups is supported (for categorial variables: Chi-square test and Fisher`s exact test, when expected cell count is freater than 5, for numerical variables: Wilcoxon test).

    • display_smd: specifies, whether the standardized mean differences should be displayed. Currently, this supports only a two-group comparison.

  • Layout specifications:

    • new_line: specifies, whether there should be an empty line after each variable

    • sort_rows: specifies the row order, in which the variables should be displayed in the Table1.

      • group_order and treatment_order: specifies the column order, in which the group categories should be displayed in the Table1.

    • add_measure_ident: specifies, whether the summary measures identifiers (e.g. n (%), mean (SD) should be added to each variable.

    • flextable_output: specifies, whether the function should return the Table 1 as flextable object, which can be instantly added to a htlm, word, latex or pdf-document. Another option included to return the Table 1 as data.frame allowing for further modifications or the applications of other tablular packages.

Table 1 for different settings

Here, the Mayo Clinic Primary Biliary Cholangitis Data from the survival package will be used for creating a Table 1.

We start with loading the required packages and the dataset in our repository. Furthermore, we get familiar with the variables in the data and doing some minor data cleaning necessary for the Table 1.

library("survival")
library(dplyr)

# Load pbc data from the survival package
pbc <- survival::pbc

baseline_var <- c("age", "chol", "stage", "platelet")

pbc[c("stage", "trt", "edema", "hepato")] <-
  lapply(pbc[c("stage", "trt", "edema", "hepato")], as.factor)

pbc <- pbc[!is.na(pbc$trt), ]
pbc$trt <- ifelse(pbc$trt == "1",
  "D-penicillamine",
  "Placebo"
)

The baseline_var vector contains all baseline variables for which we want to calculate descriptive summary measures.

Display standardized mean differences (SMD)

Default is display_smd = FALSE. This only works for pairwise group comparisons. Compared to the p-value, which depends on sample size, the standardized mean difference (SMD) does not rely on formal hypothesis testing. For numerical variables, it standardizes the difference in group means by the pooled standard deviation, whereas for categorical variables, it standardizes the difference in group proportions (or frequencies) by the pooled variance of the proportions (Austin 2011). Since the mean differences are standardized, one can now better compare differences across different scales.Generally, a SMD between -0.05 and 0.05 can be considered as well balanced.

Output options

Default setting is the output as a flextable()output, which returns a publications-ready format. This can be exported as html, pdf, world or inserted into a latex file. Alternatively, one can opt for returning the output as data.frame for additional modifications or another package for displaying the content as output table.
